# Final Review Checklist

> Last gate before delivering the document to the user.

## Step 1 — Run All Sub-Checklists
- [ ] `style-compliance-checklist.md` — all items passed
- [ ] `content-completeness-checklist.md` — all required sections present
- [ ] `docx-formatting-checklist.md` — no technical errors

## Step 2 — Placeholder Removal
- [ ] Search document text for `[` — all instances are intentional [TBD] markers
- [ ] "How To Use This Template" guidance box is NOT present in output
- [ ] Italicized guidance text removed from final output
- [ ] No generic filler text or lorem ipsum present

## Step 3 — Cross-Reference Check
- [ ] Document Control title matches cover page title exactly
- [ ] Document ID follows the correct format for the type (e.g., BRD-001)
- [ ] Version on cover matches version in Document Control table
- [ ] Date on cover matches date in Document Control table
- [ ] Related Documents list is accurate for the document type

## Step 4 — File Naming
- [ ] File name follows: `[CODE]_[ProjectName]_[YYYY-MM-DD].docx`
- [ ] No spaces in file name (use underscores)
- [ ] File size is reasonable (50KB to 5MB for text-only documents)

## Step 5 — Delivery Message to User

Include in delivery message:
- Document type and full title
- File name
- List of sections included
- Any [TBD] items the user should fill in before finalizing
- Note: "Update the Revision History table when making changes; increment version and update date."
