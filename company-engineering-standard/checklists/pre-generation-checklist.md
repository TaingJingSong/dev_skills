# Pre-Generation Checklist

> Complete this checklist BEFORE generating any document. If a required item is missing, ask the user or mark as [TBD].

## Required for All Document Types

- [ ] Document type identified (BRD / PRD / SRS / FRD / TRD / API / DD / UAT / IRL)
- [ ] Project title / name provided (or [TBD] confirmed)
- [ ] Prepared by name or department provided
- [ ] Version number provided (default: 0.1 for first draft)
- [ ] Date provided or use today's date
- [ ] Document status confirmed (default: Draft)
- [ ] Confidentiality level confirmed (default: Strictly Confidential)

## Content Inputs

- [ ] Business/project context provided (at least 1-2 sentences)
- [ ] At least one scope item (in-scope) identified
- [ ] At least one stakeholder identified
- [ ] Core requirements or features provided (at minimum 3)
- [ ] Approval contacts identified (or [TBD])

## Type-Specific Requirements

### BRD
- [ ] Business objectives (3+) provided or constructible from context

### PRD
- [ ] Product vision provided or inferable
- [ ] At least one user persona described

### SRS
- [ ] System overview available
- [ ] At least one module and its functional requirements provided

### FRD
- [ ] Module list provided
- [ ] At least one complete workflow described

### TRD
- [ ] Technology stack identified (at least partially)
- [ ] Architecture approach described

### API Spec
- [ ] At least one endpoint defined (method + URI + description)
- [ ] Authentication method identified

### Data Dictionary
- [ ] At least one entity / table defined
- [ ] At least one field definition provided

### UAT
- [ ] UAT period (from/to dates) provided or [TBD]
- [ ] At least one test case or feature to test identified

### IRL
- [ ] At least one issue OR one risk provided to populate the log

## Ready to Generate?

If 80%+ of the above are checked, proceed with generation.
If less than 80%, pause and ask the user for the missing critical inputs.
